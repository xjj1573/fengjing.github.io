Vuex

发表于2021-04-22
|阅读量:

一、什么是Vuex

Vuex 是一个专为 Vue.js 应用程序开发的状态管理模式。

它采用集中式存储管理应用的所有组件的状态，并以相应的规则保证状态以一种可预测的方式发生变化。

Vuex 也集成到 Vue 的官方调试工具 **devtools extension (opens new window)**，提供了诸如零配置的 time-travel 调试、状态快照导入导出等高级调试功能。

关键词：****状态管理 规则 状态可追踪 可预测

vuex其实就是一个数据仓库

二、为什么使用Vuex

Vuex可用于解决项目中传递复杂但共用的数据，将这些数据存储在公共的状态管理器内，供各个页面 / 组件使用。

但是Vuex不是必需的，根据项目大小及项目中状态管理的复杂程度来决定是否使用Vuex。

Vuex的功能其实可以通过父子组件传值或BUS等方式均可以解决。

四、Vuex的核心概念

1、state —— mapState

Vuex唯一的数据源，内部存放需要集中管理的状态数据

（1）数据 Store / state 中的的获取

由于状态值可能会发生变化，因此我们通常在computed中获取状态值，当state值发生变化时，computed会自动计算更新。

（2）辅助函数 —— mapState

将state中的属性值进行映射

通过mapState获取的数据可以自定义名称

2、getters —— mapGetters

可对状态数据进行筛选等进一步处理后再返回给组件

二、项目结构

Vuex 并不限制你的代码结构。但是，它规定了一些需要遵守的规则：

  1. 应用层级的状态应该集中到单个 store 对象中。
  2. 提交 mutation 是更改状态的唯一方法，并且这个过程是同步的。
  3. 异步逻辑都应该封装到 action 里面。

只要你遵守以上规则，如何组织代码随你便。如果你的 store 文件太大，只需将 action、mutation 和 getter 分割到单独的文件。
