react：用于构建用户界面的javascript库

react特点：

虚拟dom-不是真实的dom，在浏览器端js实现的一套dom api，底层通过dom diff算法，更新有差异化的部分。不会重新渲染整个页面，只更新变了的dom部分，大大提升页面渲染效率。

组件化-通过组件来开发应用。组件是能完成某个特定功能独立的可重用的代码，代码的可重用性非常高。若干个小组件构成更加复杂的UI。组件逻辑使用js编写而非模板，使得状态与dom分离。

声明式-数据确定页面就确定，帮助节约大量dom操作代码。为应用每个状态设计简洁的视图，当数据改变时react能有效地更新并正确地渲染组件。

state(状态)介绍与应用

什么是state

​ props 对外的接口 可读性

​ state 组件内部的状态 控制元素的显示隐藏 可修改

​ setState来修改

​ setState异步更新，不会立马映射更新组件，批量延迟更新 ，为了提高性能采取的一种措施。优化：多个setState调用，会进行合并 处理，连续多个set只会最后执行一次，不会导致组件重复渲染。 高效，不用考虑性能问题。把state更新对象放到一个队列里，在队列中把新的状态提取出来，再合并到state，最后触发render函数组件去重新渲染更新

​ React控制的事件处理程序（onClick,onChange） 生命周期钩子函数 ,不会同步更新state，是异步的。

setState同步步更新:react控制之外 是同步更新的

​ 原生js绑定的事件 setTimeout等

如何使用

  1. 回调 函数

2.通过对象的方式 延迟更新

如果基于当前state想计算出新的值，就要采用函数的方式，因为他不是一个对象，可以确保每次调用的都是最新的state，相当于更改完了给你一个回调。

组件的state如何划分

​ state props 最终都会影响ui展示

​ 原则

​ 1.让组件尽可能少状态 ，如果该组件只是进行ui渲染或者数据展示，没有复杂交互，应该使用props ，不使用state ，使用function 函数组件 或者。无状态组件，ui组件， 函数式组件， 纯函数。无交互，无数据逻辑层的展示。

​ 2.随着时间产生变化的数据，页面有交互， 使用state class

props与state对比

1.组件内的数据，js对象，保存信息，控制组件的形态 ui

2.props 父组件传入的，定义外部组件的接口 通过 类组件this.props或者函数内部 prop 进行使用

3.props只读的，不能直接被修改，如果想修改可以用state

4.state是组件内部的状态，作用范围仅限于当前组件，相当于私有变量。可以使用state来跟踪状态（控制元素的显示隐藏）

React 生命周期 组件

从创建到销毁的过程是生命周期

挂载阶段

constructor(props) ： 需要接收父组件的props

​ 初始化操作，state,this绑定

componentWillMount ：组件即将挂载，render之前调用，只会调用一次，很少被使用，在这里调用setState不会引起重新渲染，使用同步的setState,不会触发额外的render处理。 改名：UNSAFE_componentWillMount

16.3 版本过时 ，被标记为不安全，但可一直使用 产生副作用，或者订阅

render： 唯一必要方法，其他都可以省略，就这个不能省略。返回一个react元素(state,props) 。 不负责组件的实际渲染工作，只是返回一个ui对象的描述

真正负责渲染的是react自身的操作

注意：必须是一个纯函数，在这里不能改变state，setState不能执行有任何副作用的操作。不能跟浏览器交互，交互应放在其它生命周期函数中。

作用：1.计算props/state 返回对应的结果

  1. React.createElement 将jsx 转换为vdom对象模型

componentDidMount：组件已经挂载完成

组件挂载到dom后触发，只会调用一次，可以获取到真实的dom

适合向服务端请求数据，

1.可以保证获取到数据时组件已经处于挂载状态，直接操作dom。

2.保证在任何情况下，只能调用一次，不会发送多余的数据请求

react也可以在服务端渲染，会减少白屏时间，componentWillMount 会被调用两次，服务端调用一次，客户端调用一次。

只能在浏览器端调用，服务器端无法调用，因为服务器端无法产生dom树

调用setState会引起组件重新渲染

作用

​ 数据可以获取到，真实dom也可以获取到

​ 可以进行数据请求，进行数据修改

​ 操作真实dom，第三方库的实例化

挂载阶段挂载完后组件被渲染到页面上了

更新阶段（如果更改状态就会触发更新阶段）

componentWillReceiveProps(nextProps)组件将要接收到props

props引起的组件更新过程中触发，调用this.setState也不会触发

只要父组件的render函数被调用，无论父组件传给子组件的props有没有改变，都会触发。官方建议getDerivedStateFromProps来代替这个钩子函数

挂载阶段如果更新props是不会触发这个钩子函数

shouldComponentUpdate (nextProps,nextState)：如果返回false，render函数就不会被执行

通知react组件是否更新，有权利阻止更新，

尽量遵循默认行为，状态改变，组件就会被重新渲染。

要求必须有返回结果

可减少组件不必要的渲染，提高性能，优化

注意：不能调用setState

componentWillUpdate(nextProps,nextState)：将要更新，更新发生前执行，一般很少用，改名为： UNSAFE_componentWillUpdate

注意：不能调用setState，循环调用 相当于一个死循环，无法正常渲染。

如果要改变的话可以使用componentWillReceiveProps 或者getDerivedStateFromProps

componentDidUpdate(prevProps,prevState)更新完成调用

有机会操作dom

使用场景：判断是否发送网络请求

无论父组件props状态更改还是state状态更改都会触发该方法

卸载阶段（如果没有更改最后就会触发卸载阶段）

componentWillUnmount

做一些清理工作： 清理定时器，取消网络请求，移除事件监听等

错误处理阶段 react16版本中增加的

componentDidCatch(error,info) ：捕获错误，info详细的报错信息

getDerivedStateFromProps(props,state) 会在调用render方法之前调用，并且在初始挂载以及后续更新时都会被调用，它应返回一个对象来更新state，返回null则不更新任何内容

适用于罕见用例：state的值在任何时候都取决于props

受控组件与非受控组件

受控于非受控是针对表单而言：

受控组件

​ 依赖状态，默认值实时映射到状态->state

​ 必须使用onChange事件来进行数据更改，不然无法更改。与双向绑定类似

​ 每当表单状态发生变化时都会被写到组件的state中，叫做受控组件

优点：

​ 符合react的数据流

​ 修改使用更加方便，完全不需要获取dom

非受控组件

​ 不依赖状态，不受控制 ，获取dom，操作dom

​ 很容易与第三方组件结合
