版本差异

v4和v4之前版本比较

v4:稳定版本，大多数项目，v4属于动态路由，ex:user/1,user/2。

react-router：

react-router,路由基础库

react-router-dom,适用于浏览器环境的再次封装

react-router-native,适用于react-native环境的再次封装

react-router-config, 静态路由配置助手

拆分为的是让我们按需引入，需要哪个安装那个。

v5: 完全兼容v4，

支持react16，兼容react

使用create-react-context 实现context api

React Router常见概念

Router组件

每个router都会创建一个history对象，用来保持当前位置的追踪

两个路由对象：HashRouter （只处理静态的url），BrowerRouter Route（非静态站点，要处理不同的url）

react native ：使用MemoryHistory

Router组件

只是一个具有渲染方法的普通react组件，路由匹配成功渲染该组件

常用属性：

**path：’ ‘ ,**路由匹配规则 可以省略，字符串类型

exact：布尔类型 true 严格模式

component：要渲染的组件

render： 函数形式 逻辑操作 path匹配的时候执行

children：函数形式 逻辑操作 任何时候都能执行 匹配match就是对象{}，匹配就是null。

优先级：children>component>render

注意：如果没有指定path，无论访问什么路径，都会匹配，非严格模式下：加根路径path=”/“也是什么路径都会匹配，加上exact属性便不会都匹配。

   children无论匹没匹配到都会被渲染,render只有匹配成功才会被渲染。

Switch组件

每次最多取出一个文具

最多只能匹配一个组件，第一个匹配到了就不再往下面匹配

作用：可以将Route组件分组

​ 404页面渲染

Link与NavLink组件

声明式的可询问导航

to:string类型 {} 对象

replace:布尔 true 替换当前历史记录

NavLink：

特殊版的link，当匹配的时候可以添加样式

activeClassName activeStyle

exact属性

Redirect组件

重定向组件，to是必须的

**to:**字符串，对象

push：布尔 true 通过history.push实现

from:将要进入的url

exact： true 严格模式匹配

History对象

每个router都会创建一个history对象，用来保持当前位置的追踪

编程式导航：

push : ex：props.history.push(“/about”);

withRouter组件

**组件没经过路由匹配props会是空对象，可以用withRouter包一下，ex:**export default withRouter(App); props对象就有了

动态路由

路由规则不是预先确定的，是在渲染过程中确定的，info/1 info/2

定义： :id ，获取：props.match.params.id
